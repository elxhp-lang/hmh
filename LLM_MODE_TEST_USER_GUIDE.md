# 大模型模式测试说明（给非技术人员）

## 你需要做的：非常简单！

### 第一步：打开创意小海页面

1. 登录你的账号
2. 点击"创意小海"菜单
3. 打开对话页面

### 第二步：看到测试开关

在输入框上方，你会看到一个**模式选择区域**：

```
🔧 硬编码模式（传统）          [切换模式]
```

或

```
🧠 大模型模式（智能）          [切换模式]
```

### 第三步：测试两种模式

#### 测试硬编码模式（默认）
1. 确保显示：🔧 **硬编码模式（传统）**
2. 在输入框输入："帮我为智能水杯生成一个8秒的视频"
3. 点击发送按钮
4. 观察回复内容和脚本质量
5. 记录你的感受（回复自然吗？脚本相关吗？）

#### 测试大模型模式
1. 点击"切换模式"按钮
2. 会变成：🧠 **大模型模式（智能）**
3. 在输入框输入相同的消息："帮我为智能水杯生成一个8秒的视频"
4. 点击发送按钮
5. 观察回复内容和脚本质量
6. 记录你的感受（比硬编码模式好吗？哪里更好？）

### 第四步：测试反馈场景

#### 场景1：反馈脚本太长
1. 生成脚本后，输入："太长了"
2. 观察两种模式的响应
3. 哪一种更理解你的意思？

#### 场景2：选择脚本
1. 输入："第一个不错，就用这个"
2. 观察两种模式的响应
3. 哪一种更自然友好？

### 第五步：告诉我你的感受

回答几个简单问题：

1. **回复自然度**
   - [ ] 硬编码模式更像真人
   - [ ] 大模型模式更像真人
   - [ ] 两个都差不多

2. **脚本质量**
   - [ ] 硬编码模式的脚本更好
   - [ ] 大模型模式的脚本更好
   - [ ] 两个都差不多

3. **响应速度**
   - [ ] 硬编码模式更快
   - [ ] 大模型模式更快
   - [ ] 两个都差不多

4. **整体满意度**
   - [ ] 我更喜欢硬编码模式
   - [ ] 我更喜欢大模型模式
   - [ ] 没有明显区别

5. **你的建议**
   - 是否建议正式切换到大模型模式？
   - 有什么需要改进的地方？

## 常见问题

### Q: 切换模式会影响之前的对话吗？
A: 不会。切换模式只影响**新发送的消息**，不会影响之前的对话历史。

### Q: 如果大模型模式出问题怎么办？
A: 随时可以点击"切换模式"按钮，切回硬编码模式。

### Q: 两种模式有什么区别？
A: 
- **硬编码模式**：使用固定的规则（关键词匹配、模板回复）
- **大模型模式**：使用AI智能理解你的意思，回复更自然

### Q: 为什么要测试两种模式？
A: 因为大模型模式是新的功能，需要验证它是否真的比硬编码模式好。

### Q: 测试后必须选择一种模式吗？
A: 不用。你可以一直保持硬编码模式，或者根据需要随时切换。

## 技术细节（你可以不看）

- 硬编码模式：使用预设的关键词和规则
- 大模型模式：调用大模型API进行智能判断
- 所有操作都有详细日志，方便开发人员调试
- 如果大模型模式失败，会自动使用硬编码模式作为备用
