# 海盟会 - 应用能力说明

## 📖 应用简介

**海盟会** 是一个基于 Seedance 2.0 模型的多智能体协作视频生成平台，专注于为用户提供智能、高效、个性化的短视频创作服务。平台集成了多个专业智能体，支持用户权限管理、视频生成、财务统计、素材智能体对话等功能，帮助用户轻松创作高质量的短视频内容。

### 核心价值
- 🤖 **智能创作**：基于 AI 的视频生成，降低创作门槛
- 🎬 **多模态支持**：支持文本、图片、视频等多种输入方式
- 🎨 **个性化定制**：根据用户偏好自动调整创作风格
- 📚 **素材库管理**：学习库和商品图库，积累创作素材
- 🔄 **工作流自动化**：8步骤自动化创作流程
- 📊 **数据可视化**：能力看板、消费统计、进度追踪

## 🎯 核心能力

### 1. 智能体系统

#### 1.1 创意小海（核心智能体）
**定位：** 智能视频创作助手

**核心能力：**
- ✅ **8步骤自动化工作流**
  - 步骤1：欢迎与偏好展示
  - 步骤2：任务识别与确认
  - 步骤3：模型调用与进度反馈
  - 步骤4：脚本呈现与确认
  - 步骤5：脚本生成（3个风格）
  - 步骤6：视频生成
  - 步骤7：批量生成
  - 步骤8：结束与保存（包含用户蒸馏功能）

- ✅ **多模态输入支持**
  - 文本输入：自然语言描述
  - 图片上传：产品图片、参考图片
  - 视频上传：参考视频文件
  - 视频链接：抖音、快手、B站等多平台链接解析

- ✅ **脚本生成与修改**
  - 生成3个不同风格的脚本（简约专业、温馨生活、活力创意）
  - 支持用户反馈后重新生成
  - 保留对话历史，支持脚本对比
  - 动态调整风格（如抖音信息流直播引流）

- ✅ **视频生成**
  - 集成 Seedance 2.0 模型
  - 支持文生视频、图生视频
  - 支持视频时长自定义（4-15秒）
  - 支持多种比例（9:16、16:9、1:1等）

- ✅ **学习与进化**
  - 长期记忆系统，持续学习用户上传的文档
  - 文件学习功能（PDF、Word、Excel、TXT、Markdown等）
  - 双层能力系统（学习模式、对比模式、独立模式）
  - 能力评分与进度追踪
- 🔄 **用户蒸馏**（计划中）
  - 步骤8：结束与保存
  - 蒸馏用户偏好（风格、平台、时长等）
  - 偏好数据存储
  - 偏好可视化展示
  - 自动调整生成策略

### 2. 视频生成系统

#### 2.1 Seedance 2.0 集成
**支持的模型：**
- ✅ `doubao-seedance-2-0-260128`（标准版）
- ✅ `doubao-seedance-2-0-fast-260128`（快速版）

**生成能力：**
- ✅ 文生视频：基于文本描述生成视频
- ✅ 图生视频：基于图片生成视频
- ✅ 视频延长：延长现有视频时长
- ✅ 参考视频：使用参考视频作为风格参考
- ✅ 音频生成：自动生成背景音乐和配音

#### 2.2 视频存储
**存储方案：**
- ✅ TOS 对象存储（火山引擎 S3 兼容存储）
- ✅ 按用户隔离存储：`users/{user_id}/videos/`
- ✅ 签名 URL 生成（15分钟有效期）
- ✅ 视频持久化存储

#### 2.3 多平台视频链接解析
**支持的平台：**
- ✅ 抖音
- ✅ 快手
- ✅ B站
- ✅ 小红书
- ✅ 微博
- ✅ 更多平台持续支持

## 🎨 技术架构

### 前端技术栈
- **Framework**: Next.js 16 (App Router)
- **Core**: React 19
- **Language**: TypeScript 5
- **UI 组件**: shadcn/ui (Amber 主题)
- **Styling**: Tailwind CSS 4

### 后端技术栈
- **Framework**: Next.js 16 (App Router + API Routes)
- **Database**: Supabase (PostgreSQL + RLS)
- **Storage**: TOS (火山引擎 S3 兼容存储)
- **AI Services**: coze-coding-dev-sdk

### AI 服务
- **视频生成**: Seedance 2.0
- **大语言模型**:
  - doubao-seed-1-6-vision（视觉理解）
  - doubao-seed-1-8（多模态对话）
- **向量化**: Embedding API
- **知识库**: Knowledge API
- **文件存储**: S3 Storage API

## 📊 数据流架构

### 视频生成流程
```
用户请求
    ↓
任务识别（步骤2）
    ↓
分析参考内容（步骤3）
    ↓
生成参考脚本（步骤4）
    ↓
生成3个脚本（步骤5）
    ↓
用户选择脚本
    ↓
提交视频生成任务（步骤6）
    ↓
Seedance 2.0 生成视频
    ↓
下载视频
    ↓
上传到 TOS
    ↓
生成签名 URL
    ↓
保存到 learning_library
    ↓
返回给用户
```

### 脚本修改流程
```
用户提出修改意见
    ↓
接收反馈
    ↓
传递给 LLM
    ↓
重新生成3个脚本
    ↓
替换当前脚本
    ↓
返回新脚本
    ↓
前端保留历史
    ↓
用户对比选择
```
