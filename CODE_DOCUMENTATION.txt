# 海盟会 - 项目完整业务说明

> 本文档详细描述海盟会项目的业务逻辑、代码架构和核心流程。

---

## 一、项目概述

### 1.1 项目定位

**海盟会**是一个基于豆包 Seedance 2.0 模型的多智能体协作视频生成平台。

### 1.2 核心功能

1. **创意小海** - 智能视频创作助手，通过对话理解用户需求
2. **视频生成** - 支持文生视频、图生视频、风格参考生成
3. **学习库** - 上传优秀视频供创意小海学习
4. **商品图库** - 管理商品图片，支持多视角整合
5. **能力看板** - 展示智能体各维度的能力评分

### 1.3 技术栈

| 技术 | 说明 |
|------|------|
| Next.js 16 | 前端框架（App Router） |
| React 19 | UI 库 |
| TypeScript 5 | 类型系统 |
| Tailwind CSS 4 | 样式框架 |
| shadcn/ui | UI 组件库 |
| Supabase | 数据库 + 认证 |
| TOS | 对象存储 |

```
用户对话（创意小海）

┌─────────────────────────────────────────────────────┐

│ /api/xiaohai/agent/chat/route.ts                     │
│                                                     │
│ 1. 验证用户身份（verifyToken）                       │

│ 3. 构建 messages 数组                                │
│ 4. 调用 LLM（大模型）                                │

        ↓
┌─────────────────────────────────────────────────────┐
│ Agent 工具解析                                        │
│                                                     │
│ 模型返回 OpenAI 格式:                                │
│ { "tool_calls": [{                                  │
│     "function": {                                    │
│       "name": "submit_video_task",                   │
│       "arguments": "{\"prompt\": \"...\"}"            │
│     }                                                │
│   }] }                                               │
│                                                     │
│ route.ts 解析格式，调用对应工具                       │
└─────────────────────────────────────────────────────┘
        ↓
┌─────────────────────────────────────────────────────┐
│ Agent 工具服务                                        │
│ src/lib/agent-tools-service.ts                       │
│                                                     │
│ submitVideoTask() 函数执行：                         │
│                                                     │
│ 1. 生成 UUID 作为 videoId                           │
│ 2. 调用 SeedanceClient 创建任务                      │
│ 3. 获取 seedanceTaskId                              │
│ 4. 写入 videos 表（必须！）                          │
│ 5. 返回 video_id 和 seedance_task_id                │
└─────────────────────────────────────────────────────┘
        ↓
┌─────────────────────────────────────────────────────┐
│ Seedance 2.0 API                                     │

│                                                     │
│ 三种模式：                                           │
│ - 文生视频：text_to_video                            │

│                                                     │
│ 调用后返回 task_id（不等待完成）                     │
└─────────────────────────────────────────────────────┘
        ↓
┌─────────────────────────────────────────────────────┐
│ 前端轮询                                             │
│                                                     │
│ 前端定时请求 /api/video/generate?taskId=xxx          │
│                                                     │
│ 查询 videos 表中的任务状态                           │
│ 返回：status, video_url（如果有）                    │
└─────────────────────────────────────────────────────┘
        ↓
┌─────────────────────────────────────────────────────┐
│ 视频完成                                             │
│                                                     │
│ status = 'succeeded'                                │
│                                                     │
│ 用户获取视频：                                       │
│ GET /api/material/history?type=personal             │
└─────────────────────────────────────────────────────┘
```

### 2.2 数据库关键表

#### videos 表（视频生成记录）
```sql
CREATE TABLE videos (
  id UUID PRIMARY KEY,           -- 我们的 video_id
  task_id TEXT,                  -- Seedance 的 task_id
  user_id UUID,                  -- 用户 ID（必须正确！）
  prompt TEXT,                   -- 视频描述
  task_type TEXT,                -- text_to_video / image_to_video / multi_modal
  status TEXT,                   -- processing / succeeded / failed
  ratio TEXT,                    -- 视频比例 9:16 / 16:9
  duration INTEGER,              -- 时长（秒）
  model TEXT,                    -- 模型 ID
  tos_key TEXT,                  -- TOS 存储路径
  result_url TEXT,               -- 视频 URL
  error_message TEXT,            -- 错误信息
  created_at TIMESTAMP
);
```

#### creative_memories 表（创意记忆）
```sql
CREATE TABLE creative_memories (
  id UUID PRIMARY KEY,
  user_id UUID,
  memory_type TEXT,              -- user_preference / learning / workflow
  content JSONB,                 -- 记忆内容
  keywords TEXT[],               -- 关键词（用于搜索）
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

#### learning_library 表（学习库）
```sql
CREATE TABLE learning_library (
  id UUID PRIMARY KEY,
  user_id UUID,
  video_name TEXT,
  video_url TEXT,                -- 视频 URL
  source TEXT,                   -- local / douyin / kuaishou / bilibili
  analysis JSONB,                 -- 视频分析结果
  keywords TEXT[],
  created_at TIMESTAMP
);
```

#### product_library 表（商品图库）
```sql
CREATE TABLE product_library (
  id UUID PRIMARY KEY,
  user_id UUID,
  product_name TEXT,
  product_description TEXT,
  category TEXT,
  tags TEXT[],
  images JSONB,                  -- 多视角图片数组
  primary_image_index INTEGER,
  usage_count INTEGER DEFAULT 0,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);
```

## 三、核心模块详解

### 3.1 创意小海 Agent

#### 文件位置
- API: `src/app/api/xiaohai/agent/chat/route.ts`
- 工具服务: `src/lib/agent-tools-service.ts`
- 系统提示词: `src/lib/xiaohai-system-prompt-v3.ts`

#### 工作流程

```
用户发送消息
        ↓
/api/xiaohai/agent/chat
        ↓
1. verifyToken() 验证身份
        ↓
2. 构建 messages 数组
   - system 消息（系统提示词）
   - 历史对话
   - 用户新消息
        ↓
3. 调用 LLM (doubao-seed-2-0-pro-260215)
        ↓
4. 解析 LLM 返回
   - 如果是普通文本 → 直接返回
   - 如果是工具调用 → 执行工具
        ↓
5. 工具执行结果返回给 LLM
        ↓
6. LLM 整合后返回最终响应
```

#### 14 个工具

| 工具名 | 功能 |
|--------|------|
| getUserPreference | 获取用户偏好设置 |
| saveUserPreference | 保存用户偏好 |
| analyzeVideo | 分析视频内容 |
| generateScript | 生成视频脚本 |
| submitVideoTask | 提交视频生成任务 |
| queryTaskStatus | 查询任务状态 |
| getTemplates | 获取脚本模板 |
| batchGenerate | 批量生成视频 |
| searchProduct | 搜索商品参考 |
| analyzeImage | 分析图片 |
| analyzeMultipleImages | 分析多张图片 |
| generateFirstFrame | 生成首帧参考图 |
| getLearningLibrary | 获取学习库内容 |
| getProductReference | 获取商品参考 |

#### 核心代码：submitVideoTask

```typescript
async submitVideoTask(userId, prompt, firstFrameUrl, duration, options) {
  // 1. 生成 UUID
  const videoId = uuidv4();
  
  // 2. 调用 Seedance
  const createResponse = await seedance.createTask({
    model: 'doubao-seedance-2-0-260128',
    content: [...],
    duration: 8,
    ratio: '9:16'
  });
  const seedanceTaskId = createResponse.id;
  
  // 3. 写入 videos 表（必须！）
  await supabase.from('videos').insert({
    id: videoId,
    task_id: seedanceTaskId,
    user_id: userId,  // ← 必须是真实 userId
    prompt,
    status: 'processing'
  });
  
  // 4. 返回
  return { video_id: videoId, seedance_task_id: seedanceTaskId };
}
```

### 3.2 视频生成服务

#### 文件位置
- Seedance 客户端: `src/lib/seedance-client.ts`
- 轮询服务: `src/lib/video-generation-poller.ts`

#### Seedance 客户端

```typescript
class SeedanceClient {
  // 创建任务（不等待完成）
  async createTask(params: {
    model: string;
    content: Array<{type: string; text?: string; image_url?: any}>;
    duration: number;
    ratio: string;
    reference_videos?: any[];
    generate_audio?: boolean;
  }) {
    // 调用豆包 API
    const response = await fetch('https://ark.cn-beijing.volces.com/api/v3/...', {
      method: 'POST',
      headers: { 'Authorization': `Bearer ${ARK_API_KEY}` },
      body: JSON.stringify({ model, messages: [{role: 'user', content: params.content}] })
    });
    return { id: response.task_id, ... };
  }
  
  // 查询任务状态
  async getTask(taskId: string) {
    const response = await fetch(`.../${taskId}`);
    return {
      status: response.status,  // processing / succeeded / failed
      result: response.output?.video_url
    };
  }
}
```

#### 三种视频生成模式

| 模式 | 参数 | 说明 |
|------|------|------|
| 文生视频 | prompt | 纯文本生成视频 |
| 图生视频 | prompt + firstFrameUrl | 图片作为首帧生成视频 |
| 风格参考 | prompt + referenceVideo | 参考视频风格生成 |

### 3.3 用户认证

#### 流程

```
用户登录
        ↓
POST /api/auth/login
{
  username: "test1",
  password: "test123"
}
        ↓
查询 users 表，验证 bcrypt 密码
        ↓
生成 JWT token
{
  userId: "xxx",
  username: "test1",
  role: "material_member"
}
        ↓
返回 token 给前端
        ↓
前端存储 token，后续请求携带
Authorization: Bearer <token>
```

#### 相关文件
- 登录 API: `src/app/api/auth/login/route.ts`
- JWT 验证: `src/lib/auth.ts`
- 注册 API: `src/app/api/auth/register/route.ts`

### 3.4 文件存储

#### TOS 对象存储

```
用户上传文件
        ↓
POST /api/upload
        ↓
1. 验证 token
2. 生成唯一文件名
3. 上传到 TOS
   路径格式: users/{userId}/images/{filename}
        ↓
4. 返回 fileKey
```

#### 视频存储

```
视频生成完成
        ↓
Seedance 返回 video_url
        ↓
下载视频到 TOS
   路径: users/{userId}/videos/{videoId}.mp4
        ↓
更新 videos 表
   tos_key = "users/xxx/videos/xxx.mp4"
        ↓
用户访问时生成签名 URL
```

## 七、常见问题

### 7.1 视频生成失败

**排查步骤**:
1. 检查 videos 表记录是否存在
2. 检查 user_id 是否正确
3. 检查 Seedance API 是否可用
4. 查看 videos 表的 error_message 字段

### 7.2 用户看不到视频

**原因**: user_id 为空或 '00000000-0000-0000-0000-000000000000'

**解决方案**:
```sql
-- 查找 user_id 为空的记录
SELECT * FROM videos WHERE user_id IS NULL OR user_id = '00000000-0000-0000-0000-000000000000';

-- 更新为正确的 user_id
UPDATE videos SET user_id = '正确的user_id' WHERE id = 'video_id';
```

### 7.3 Token 过期

登录页使用"记住密码"后 token 会长期有效，但某些测试 token 会过期。

## 九、红线规则

### 9.1 submitVideoTask 必须写 videos 表

```typescript
// ✅ 正确
await supabase.from('videos').insert({
  id: videoId,
  task_id: seedanceTaskId,
  user_id: userId,  // 必须用真实 userId
  status: 'processing'
});

// ❌ 错误：只调用 API 不写表
const taskId = await seedance.createTask(...);
return { taskId };  // 任务无法追踪！
```

### 9.2 userId 必须正确

- 不能是 '00000000-0000-0000-0000-000000000000'
- 不能是 NULL
- 必须是从 token 中解析出的真实 userId

### 9.3 不要删除核心业务链

```
用户对话 → Agent 工具 → videos 表 → Seedance API → TOS 存储
```

这个链路不能断，任何修改都要确保链路完整。
