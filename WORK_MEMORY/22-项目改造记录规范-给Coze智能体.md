# 项目改造记录规范（给 Coze 侧智能体）

> 目的：在代码回传 Coze 后，让后续智能体快速理解本轮改造的背景、行为与关键技术决策。  
> 使用方式：每次阶段提交后，必须更新本文件。

---

## 1. 改造背景

- 项目从原型态升级为可交付产品态。
- 本轮重点：素材中心合并、创意链路稳定化、真人素材库、Remix、模板库联动。
- 兼容要求：保持现有 Coze 运行规则与核心接口兼容。

---

## 2. 本轮关键决策（不可丢失）

1. 素材历史 + 商品库合并为单一「素材中心」页面。
2. 首页仅以窗口方式调用素材中心。
3. 真人素材库作为素材中心子页（演员素材）。
4. 调用真人素材时：
   - 传 `asset://{asset_id}` 给生成链路
   - `asset_url` 只用于前端预览
5. REMIX 必须支持：原视频 + 原脚本 + 原参数继承再生成。
6. 取消全局助手与高风险三期需求。

## 4. 数据结构约定（本轮新增）

### 4.1 real_assets（演员素材）
- `id`
- `asset_id`（唯一）
- `asset_url`（预览）
- `name`
- `description`
- `category`
- `status`
- `created_at`
- `updated_at`

### 4.2 视频任务扩展字段（用于 Remix）
- `prompt_snapshot`
- `params_snapshot`
- `reference_snapshot`

- `source_type`（agent/manual/remix）

## 5. API 约定（本轮新增/增强）

### 5.1 真人素材
- `GET /api/real-assets`
- `GET /api/real-assets/:id`
- `POST /api/real-assets`（管理员）
- `PUT /api/real-assets/:id`（管理员）
- `DELETE /api/real-assets/:id`（管理员）

### 5.2 Remix
- `POST /api/video/remix`
  - 入参：`sourceVideoId` + `overrides?`
  - 出参：新任务信息 + 源任务关联
