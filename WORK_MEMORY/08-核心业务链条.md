## 三、红线清单（绝对不能碰！）

### ❌ 红线 1：AgentToolsService.submitVideoTask 必须写 videos 表

**错误做法**：
```typescript
// ❌ 错误：只返回结果，不写表
async generate_video(params) {
  const taskId = await seedance.createTask(...);
  return { video_id, seedance_task_id };
}
```

**正确做法**：
```typescript
// ✅ 正确：必须写入 videos 表
async submitVideoTask(userId, prompt, ...) {
  const seedanceTaskId = await seedance.createTask(...);
  await supabase.from('videos').insert({
    id: videoId,
    task_id: seedanceTaskId,
    user_id: userId,  // ⚠️ 必须用真实的 userId
    status: 'processing'
  });
  return { video_id, seedance_task_id };
}
```

### ❌ 红线 2：userId 必须正确传递和保存

**原因**：如果 user_id 是默认值 `00000000-0000-0000-0000-000000000000`：
- 用户看不到自己生成的视频
- 无法统计消费
- 无法发送通知

**验证方法**：
```sql
SELECT id, user_id, prompt FROM videos ORDER BY created_at DESC LIMIT 5;
-- user_id 必须是真实用户，不是 00000000...
```

### ❌ 红线 3：route.ts 必须兼容 OpenAI 工具调用格式

**模型返回格式**：
```json
{
  "tool_calls": [{
    "type": "function",
    "function": {
      "name": "submit_video_task",
      "arguments": "{\"prompt\": \"...\", \"duration\": 5}"
    }
  }]
}
```

**route.ts 必须解析这个格式**：
```typescript
// 格式3: OpenAI 格式
if (parsed.tool_calls && Array.isArray(parsed.tool_calls)) {
  for (const tc of parsed.tool_calls) {
    let params = {};
    if (tc.function?.arguments) {
      params = JSON.parse(tc.function.arguments);
    }
    toolCalls.push({
      name: tc.function?.name,
      params: params
    });
  }
}
```


```sql
SELECT id, user_id FROM videos WHERE user_id = '00000000-0000-0000-0000-000000000000';
-- 如果有结果，说明有问题
```

### 检查 token 是否过期
```bash
# 解码 JWT token 的 exp 字段
date -d @<exp_timestamp>
# 如果日期在过去，说明 token 已过期
```
