# 创意小海板块

## ⭐ 核心业务路径（必须牢记）

```
用户打开创意小海对话框（核心入口）
    ↓
【核心操作】发送参考视频（链接/上传）
    ↓
AI 分析视频风格 → 提取 reference_video 参数
    ↓
可选：上传商品图片 → AI 分析商品特征
    ↓
生成脚本（英文提示词 + 多模态参数）
    ↓
【新增】生成首帧图（脚本 + 商品图 → 图生图）
    ↓
调用 Seedance 2.0 → 生成同风格视频
    ↓
用户预览/下载
```

**图片只是辅助**：用于首帧参考，不是主要入口

## Seedance 2.0 完整能力 ⭐⭐⭐

### 1. 生成模式
| 模式 | 用途 | 核心参数 |
|------|------|----------|
| **文生视频** | 仅文本提示词生成 | `text` |
| **图生视频** | 首帧图片 + 文本 | `first_frame` + `text` |
| **多模态参考** | 核心能力！风格/动作参考 | `reference_video/image/audio` |
| **视频编辑** | 基于原视频 + 编辑指令 | `reference_video` + `text` |
| **视频延长** | 1-3个视频延长 | `reference_video` (最多3个) |

### 2. 多模态参数详解
```typescript
// 首帧图片 - 视频开始画面
{ type: 'image_url', image_url: { url }, role: 'first_frame' }

// 尾帧图片 - 视频结束画面
{ type: 'image_url', image_url: { url }, role: 'last_frame' }

// 参考图片 - 风格/内容参考
{ type: 'image_url', image_url: { url }, role: 'reference_image' }

// 参考视频 - 风格/动作参考 ← 【核心】
{ type: 'video_url', video_url: { url }, role: 'reference_video' }

// 参考音频 - 背景音乐
{ type: 'audio_url', audio_url: { url }, role: 'reference_audio' }
```

### 3. 参数配置
| 参数 | 可选值 | 默认值 |
|------|--------|--------|
| `duration` | 4-15秒 | 5秒 |
| `ratio` | 16:9, 9:16, 1:1, 4:3, 3:4, 21:9, adaptive | 16:9 |
| `generate_audio` | true/false | true |
| `watermark` | true/false | false |

### 4. 模型
| 模型 ID | 名称 | 特点 |
|---------|------|------|
| `doubao-seedance-2-0-260128` | Seedance 2.0 标准版 | 质量优先 |
| `doubao-seedance-2-0-fast-260128` | Seedance 2.0 快速版 | 速度优先 |

### 5. ⚠️ 限制（必须牢记）
- **不支持真人人脸** - 提示词避免描述真人
- **需要英文提示词** - prompt 必须是英文
- **时长限制** - 4-15秒，超出自动截断

### 6. 同风格生成实现
```json
{
  "reference_video": "用户上传的参考视频URL",
  "reference_mode": "style+motion",
  "reference_audio": "从参考视频中提取的音频URL（可选）"
}
```

## 8阶段工作流

### 阶段定义
```
1. input_collecting   - 收集参考视频（核心）
2. analyzing          - 分析视频风格
3. script_generating  - 生成详细分镜脚本
4. script_confirming  - 用户确认或修改脚本
5. video_generating   - 根据脚本生成视频
6. reviewing          - 用户审核视频
7. batch_preparing    - 生成修改方向
8. batch_generating   - 批量生成视频
```

## ⭐ 首帧图生成功能（新增）

### 功能说明
根据商品图和脚本描述，生成符合视频开头场景的首帧图。
支持图生图模式，保持商品特征。

### 工具参数
```typescript
generate_first_frame: {
  product_image_url: string;      // 商品图片 URL（必需）
  script_content: string;          // 脚本描述
  aspect_ratio?: string;          // 9:16 或 16:9，默认 9:16
  style?: string;                 // 风格描述
  reference_style?: string;       // 参考视频风格
}
```

### 返回值
```json
{
  "success": true,
  "data": {
    "image_url": "生成的首帧图 URL",
    "image_id": "图片 ID"
  }
}
```

### 完整工作流
```
用户上传商品图 → analyze_image 分析
    ↓
用户选择参考视频 → analyze_video 分析风格
    ↓
generate_script 生成3个脚本
    ↓
用户选择脚本
    ↓
generate_first_frame 生成首帧图（商品图 + 脚本 + 参考视频风格）
    ↓
用户确认首帧图
    ↓
submit_video_task 提交视频生成（prompt + first_frame_url + reference_video）
    ↓
query_task_status 轮询结果
```

### SDK 能力
使用 `ImageGenerationClient`（coze-coding-dev-sdk）：
- 文生图 (text-to-image)
- 图生图 (image-to-image)
- 支持 2K/4K 尺寸

## ⚠️ 问题排查

### 视频上传错误
错误：`Unexpected token '<', "<!DOCTYPE "... is not valid JSON`

原因分析（来自 Seed 2.0 Pro）：
1. **服务端路由/权限问题** - 上传接口未正确配置、用户未登录/令牌过期
2. **服务器内部错误** - 文件过大超出限制、存储服务故障
3. **请求格式错误** - Content-Type 设置错误、参数缺失
4. **环境配置问题** - 接口地址配置错误

排查建议：
- 检查上传 API 返回的是 JSON 还是 HTML 错误页面
- 验证用户认证状态
- 检查文件大小限制
- 查看服务端日志

### 2025-04-07 视频上传流程优化（第二次）
- ✅ **区分上传入口**：学习库 vs 创意小海
- ✅ **URL 时效差异**：学习库永久 URL，创意小海临时 URL（1小时）
- ✅ **纯 Agent 架构**：工具定义改为"能力清单"，不强制流程

#### 上传入口区分

| 入口 | API | URL 时效 | 后续处理 |
|-----|-----|---------|---------|
| 学习库 | `/api/learning-library/upload` | 永久（expiresIn=0） | 存储数据，不激活 Seed 2.0 |
| 创意小海 | `/api/creative-agent/upload` | 临时（3600秒） | 直接发给 Seed 2.0 进行意图识别 |

#### 纯 Agent 架构原则

**我们只提供信息，不强制执行：**

1. **工具定义 = 能力清单**，不是"强制流程"
   ```
   ❌ 旧：当你收到视频时，必须先调用 analyze_video 工具
   ✅ 新：你有以下工具可用，你决定何时使用
   ```

2. **信息传递，非命令**
   ```
   ❌ 旧：必须分析这个视频
   ✅ 新：用户上传了一个视频，你有这些能力
   ```

3. **Seed 2.0 自由决策**
   ```
   - 它决定是否调用工具
   - 它决定调用哪些工具
   - 它决定如何组合工具
   ```

#### 新增 API

- `POST /api/learning-library/upload` - 学习库上传（永久 URL）
- `POST /api/learning-library/upload/confirm` - 学习库上传确认
- `POST /api/creative-agent/upload` - 创意小海上传（临时 URL）
- `POST /api/upload/presign` - 通用预签名上传（支持 expiresIn 参数）

#### 流程对比

```
学习库上传：
用户上传视频
    ↓
存储到 TOS（永久公开 URL）
    ↓
创建数据库记录
    ↓
通知 Seed 2.0："有新的偏好数据"（由它决定是否学习）
    ↓
用户可在创意小海对话中查看学习库内容

---

创意小海上传：
用户上传视频
    ↓
存储到 TOS（临时公开 URL，1小时）
    ↓
返回临时公开 URL
    ↓
前端将 URL 发给 Seed 2.0
    ↓
Seed 2.0 自主决定如何处理：
  - 分析视频内容
  - 询问用户意图
  - 生成类似视频
  - 保存到学习库
```
