---

## 一、背景

### 1.1 现有问题
- 素材助手与创意小海功能重叠
- 财务助手、权限助手未实现
- 需要新增运维助手功能

### 1.2 目标
简化智能体系统，减少冗余，增加实用的运维能力

---

## 二、方案详情

### 阶段一：删除功能（低风险）

#### 2.1.1 删除素材助手
- **原因**：与创意小海功能重叠
- **影响**：仅影响 `/agent` 页面的切换
- **风险**：低

#### 2.1.2 删除权限助手
- **原因**：功能未实现
- **影响**：仅影响 `/agent` 页面的切换
- **风险**：低

**预估时间**：30分钟

---

### 阶段二：财务助手（中等复杂度）

#### 2.2.1 功能设计
| 功能 | 说明 |
|------|------|
| 账单查询 | 展示用户消费记录 |
| 消费统计 | 按月/季度/年统计 |
| 费用分析 | AI 分析消费趋势 |
| 发票申请 | 申请开票功能 |

#### 2.2.2 技术方案
| 项目 | 技术选型 |
|------|----------|
| 主模型 | doubao-seed-1-8-251228 |
| 对话模式 | 纯 Agent（直接 LLM 对话） |
| 数据展示 | 轮询系统（定时拉取数据） |
| 数据来源 | billing 表 + 需确认 |

#### 2.2.3 注意事项
⚠️ 需要确认财务数据来源（目前只有 8 条测试数据）

**预估时间**：2-3小时

---

### 阶段三：运维助手（高复杂度，需谨慎）

#### 2.3.1 功能设计
| 功能 | 可行性 | 说明 |
|------|--------|------|
| 定时巡检 | ✅ 可行 | 定时扫描代码 |
| 飞书告警 | ✅ 可行 | 错误告警到运维群 |
| 自动修复 | ⚠️ 不推荐 | 风险太高 |

#### 2.3.2 推荐方案（保守）
| 阶段 | 功能 | 说明 |
|------|------|------|
| P1 | 飞书告警 | 用户报错 → 飞书通知运维 |
| P2 | 定时巡检 | 定时检查代码，生成报告 |
| P3 | AI 分析 | AI 分析错误原因，提供建议 |

#### 2.3.3 不推荐功能
❌ 自动修复代码（风险太高，可能引入更多问题）

**预估时间**：3-4小时（保守方案）

  await fetch(FEISHU_WEBHOOK_URL, {

  });
}
