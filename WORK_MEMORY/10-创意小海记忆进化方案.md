# 创意小海记忆与进化系统 - 完整方案

**创建日期**: 2026-04-15
**版本**: v1.1
**状态**: 已完成（简化版，无置信度）

---

## 一、需求回顾

用户想要的创意小海能力：
1. **记忆能力** - 分用户记忆，不固化
2. **自我进化能力** - 从错误/反馈中学习
3. **文件学习能力** - 学习上传的文档

---

## 二、设计原则

| 原则 | 说明 |
|------|------|
| 不固化 | 记忆不导致回复固化，LLM保持灵活性 |
| 不限制数量 | 按相关性返回，不人为限制 |
| 自主决策 | 记忆是否保存由LLM自行判断 |
| 用户隔离 | 不同用户记忆完全隔离 |
| 简单排序 | 按关键词命中数排序，不加置信度 |

## 四、数据库设计

### 4.1 用户记忆表：xiaohai_user_memories

```sql
CREATE TABLE xiaohai_user_memories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL,
  content TEXT NOT NULL,
  memory_type VARCHAR(50) DEFAULT 'general',
  keywords TEXT[] DEFAULT '{}',
  source VARCHAR(50) DEFAULT 'user',
  metadata JSONB DEFAULT '{}',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_xiaohai_memories_user_id ON xiaohai_user_memories(user_id);
CREATE INDEX idx_xiaohai_memories_keywords ON xiaohai_user_memories USING GIN(keywords);
```

### 4.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 用户ID，用于隔离 |
| content | TEXT | 记忆内容 |
| memory_type | VARCHAR | 类型：general/preference/experience/rule/document |
| keywords | TEXT[] | 关键词数组 |
| source | VARCHAR | 来源：user/feedback/error/document |
| metadata | JSONB | 扩展元数据 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

### 4.3 学习记录表：xiaohai_learning_records

```sql
CREATE TABLE xiaohai_learning_records (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID,
  record_type VARCHAR(50) NOT NULL,
  content TEXT NOT NULL,
  original_content TEXT,
  feedback TEXT,
  score INT,
  tags TEXT[],
  source VARCHAR(50) DEFAULT 'user_feedback',
  status VARCHAR(50) DEFAULT 'active',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_learning_records_user_id ON xiaohai_learning_records(user_id);
CREATE INDEX idx_learning_records_type ON xiaohai_learning_records(record_type);
```

## 六、排序机制

### 按关键词命中数排序

```typescript
// 提取查询关键词
const queryKeywords = extractKeywords(query)

// 计算命中数
const hits = item.keywords.filter(k => 
  queryKeywords.some(qk => k.includes(qk) || qk.includes(k))
).length

// 排序：按命中数降序
.sort((a, b) => b.hits - a.hits)
```

**注意**：不使用置信度字段，排序简单直接。

## 十一、注意事项

1. **Skills vs Tools 区别**：
   - Skills：给AI Coding Agent用的技能
   - Tools：给智能体用的工具

2. **用户隔离**：所有记忆和记录按 user_id 隔离

3. **简化原则**：不需要置信度等复杂机制，按关键词命中排序即可
