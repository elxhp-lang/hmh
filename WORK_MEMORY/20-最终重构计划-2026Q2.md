# 最终重构计划（2026Q2）

> 目标：在保持 Coze 可部署与现有数据兼容前提下，完成全产品重构与功能升级。  
> 状态：已合并最新决策（素材中心合并、REMIX、真人素材库、模板库保留）。

---

## 1. 已确认的产品决策

1. 素材历史与素材仓库合并为一个页面（统一称「素材中心」）。
2. 首页通过窗口形式调用素材中心，不再重复建设独立素材管理页。
3. 保留 REMIX 功能：基于原视频 + 原脚本 + 原参数回传 Seedance 2.0 再生成。
4. 取消全局助手浮窗与原第三期高风险内容。
5. 模板库保留，并接入创意小海与手动视频页。
6. 商品库升级为素材库，子页面展示：
   - 商品素材
   - 演员素材（真人素材库）
7. 真人素材调用规则：
   - 生成参数使用 `asset://{asset_id}`
   - `asset_url` 仅用于预览展示演员素材

---

## 2. 最终范围（In Scope）

### A. 首页/概览页
- 统计卡（生成中/完成/失败/总量）+ 跳转
- 快捷入口（创意小海、手动视频、素材中心）
- 以弹窗/侧栏方式打开素材中心（默认最近7天）

### B. 创意小海
- SSE 消息协议类型化渲染（文本/脚本/分析/任务状态）
- 提示词辅助（优化、改写、扩写）
- 模板库套用

### C. 手动视频页
- 保持 Seedance 2.0 直接调用
- 与创意小海共享参数校验与任务落库逻辑

### D. 素材中心（统一页）
- 商品素材管理
- 演员素材管理（人脸库展示）
- 搜索/筛选/标签/批量操作
- 历史记录、下载、删除、分享
- REMIX 一键再生成

### E. 真人素材库（real_assets）
- 管理员 CRUD
- 用户可见列表（active）
- 生成调用拼接 `asset://{asset_id}`
- UI 使用 `asset_url` 展示预览图

---

## 3. 兼容性原则（必须遵守）

1. 不改 Coze 运行入口：
   - `.coze`
   - `scripts/dev-start.sh`
   - `scripts/build.sh`
   - `scripts/start.sh`
2. 旧路由尽量兼容（必要时重定向，不直接移除）。
3. 旧表结构不破坏，新增字段与新表采用向后兼容策略。
4. 外部 API 保持同路径同主要字段，新增字段采用可选方式。

---

## 4. 技术实施分阶段

### Phase 0：基线与契约冻结
- 冻结关键接口契约（auth/video/material/agent）
- 建立变更准则：先兼容，后替换

### Phase 1：创意链路稳定化
- 重构 `api/xiaohai/agent/chat` 的编排层与结构化输出
- 前端创意页改为类型驱动渲染

### Phase 2：素材中心合并
- 合并商品库与素材历史能力
- 首页窗口调用素材中心
- 路由兼容跳转

### Phase 3：真人素材库落地
- 新建 `real_assets` 数据表
- API：`/api/real-assets`（用户读、管理员写）
- 素材中心新增「演员素材」子页
- 生成链路支持 `asset://{asset_id}`

### Phase 4：REMIX 闭环
- `POST /api/video/remix`
- 继承源任务脚本/参数/引用素材
- 新任务与源任务关联（parent_video_id/source_type）

### Phase 5：模板库联动与回归验收
- 模板库在创意小海/手动页统一套用
- 端到端回归：生成、素材、remix、权限、部署
