# 业务冒烟验收清单（按角色）

## 目标

验证“业务逻辑可跑通”而不是只看编译通过。每项按 `通过/失败 + 截图或日志` 记录。

---

## A. 通用前置

1. 成功登录并进入 `dashboard`
2. 网络正常，TOS/Seedance 可访问
3. 数据库可写（新建记录后可查到）
4. 执行：
   - `pnpm run ts-check`
   - `pnpm exec eslint "src/**/*.{ts,tsx}"`
   - `pnpm next build`
   - `pnpm run contract-check`

---

## B. 素材成员（material_member）

### B1. 创意小海主链路

1. 进入 `agent/create`，发送文本生成请求
2. 观察任务栏出现任务（running）
3. 等待结果，任务应转为终态（succeeded/failed）
4. 点击任务回放，聊天区应显示回放内容
5. 用同一 `task_id` 让 Agent 查询状态，结果应与任务栏一致

### B2. 视频页链路

1. 进入 `video`，提交文生视频任务
2. 刷新页面后历史仍能看到任务记录
3. 任务完成后可在历史中预览/打开视频 URL
4. 删除历史记录后，列表同步减少

### B3. 学习库链路

1. 上传视频（任一入口：普通/分块/上传确认）
2. 记录应写入 `learning_library`，`analysis_status` 进入 pending/processing
3. 分析完成后变为 completed 或失败并含错误信息
4. `video_key` 前缀应为 `users/{userId}/learning-videos/`（legacy 数据允许旧前缀）

---

## C. 素材组长（material_leader）

### C1. 素材中心

1. 进入 `material/history`，切换团队与个人视图
2. 搜索、状态过滤、标签过滤应生效（结果集变化符合预期）
3. REMIX 任务提交后，能在任务链路中追踪到新任务
4. 标签池新增/删除、视频标签编辑后刷新仍一致

---

## D. 财务/管理员（finance/admin/super_admin）

### D1. 账单与导出

1. 进入 `billing`，筛选后列表与总额口径一致
2. 导出 CSV 字段完整（包含用户信息且不为空）

### D2. TOS 管理接口鉴权

1. 普通用户调用 `/api/tos/sign|acl|cors` 返回 403
2. 管理员调用上述接口返回 200

### D3. 用户审批

1. `admin/users` 审批用户后状态更新正确
2. 拒绝用户后登录受限（提示等待审核或被禁用）

---

## E. 关键一致性断言（必须全绿）

1. 任务状态一致：UI任务栏 = Agent查询 = DB状态
2. ID一致：返回 `task_id` 可直接用于后续查询，不需手工猜测切换
3. 查询语义一致：过滤条件与统计总数匹配
4. 学习库路径一致：新写入 key 使用统一前缀
5. 异常可见：失败时有错误信息，不出现“无声失败”

---

## F. 失败处理规则

若任一条失败：

1. 记录页面、接口、请求参数、返回值
2. 补充数据库快照（关键字段）
3. 标记为 P0/P1/P2
4. 修复后回归重跑本清单，不允许“只修不回归”
