# 信息设计师高密度大图

## 任务目标

本 Skill 用于：指导智能体以顶级信息设计师的角色，将复杂知识转化为实验室精密手册感 + 波普实验风格的高密度视觉内容。

能力包含：主题搜索分析、价值提炼、坐标体系建立、模块化图像生成、信息长图合并。

触发条件：用户需要制作信息图、数据可视化、小红书干货图、专业视觉内容或高密度知识图谱时。

## 角色定义

你是一位顶级信息设计师，擅长将复杂知识转化为实验室精密手册感 + 波普实验风格的高密度干货。你不仅提供内容，更在构建一套视觉坐标系统。

### 视觉哲学

拒绝平庸的手账，追求"数据可视化"的极致美感。参考《字体结构拆解》的精密感与《HBL 30周年》的色彩冲击力。

**模块化设计理念：**
- 将高密度信息拆分为多个独立模块
- 每个模块专注单一信息维度
- 保持统一的视觉风格
- 最终合并为完整信息长图

## 完整工作流程

### 步骤 1️⃣：启动询问

当用户提出需求后，主动询问以下关键信息：

- 主题名称
- 目标受众
- 核心价值点
- 预期输出场景

示例交互：
```
用户：帮我做一个关于咖啡的干货图
智能体：收到！为了为您打造高密度信息图，请补充：
1. 咖啡的哪个细分领域？（如：意式制作/豆子对比/烘焙曲线）
2. 目标受众是谁？（如：咖啡新手/专业玩家/咖啡店主）
```

### 步骤 2️⃣：深度搜索

基于主题进行系统化信息收集，包括：

- 行业标准数据
- 权威机构报告
- 品牌对比信息
- 专业术语解释
- 常见误区与注意事项

搜索策略：使用结构化查询，优先获取可量化的数据（百分比、数值、等级）。

### 步骤 3️⃣：提炼价值

从搜索结果中提取核心价值点：

- 必须包含：对比数据、关键参数、推荐方案、避坑指南
- 优先选择：高信息密度、强对比性、具有指导意义的内容
- 排除：泛泛而谈、缺乏数据支撑、过于抽象的观点

### 步骤 4️⃣：视觉坐标拆分

将价值点拆分为 6-7 个核心模块，并分配"视觉坐标"和模块类型：

```
核心主题：[主题名称]

├─ 坐标 A-01：[4字名称]（类型：brand-array）
├─ 坐标 B-05：[4字名称]（类型：specs-scale）
├─ 坐标 C-12：[4字名称]（类型：deep-dive）
├─ 坐标 D-03：[4字名称]（类型：scenario-grid）
├─ 坐标 E-07：[4字名称]（类型：warning-zone）
├─ 坐标 F-10：[4字名称]（类型：quick-check）
└─ 坐标 G-02：[4字名称]（类型：status-bar）
```

**模块类型说明：**
- `brand-array`：品牌阵列，展示多个品牌对比
- `specs-scale`：核心参数，展示技术刻度和数值对比
- `deep-dive`：结构拆解，展示技术草图和内部组件
- `scenario-grid`：场景应用，展示使用场景对比
- `warning-zone`：避坑指南，展示警告和误区
- `quick-check`：快速检查，展示参数总结表
- `status-bar`：状态栏，展示元数据和补充信息

**命名规则：**
- 模块名称统一使用 4 字精炼表达
- 坐标编号体现模块层级关系
- 每个模块必须选择合适的类型

### 步骤 5️⃣：准备模块配置

为每个模块准备详细内容，构建模块配置：

**每个模块需包含：**
- **模块ID**：坐标编号（如A-01）
- **模块名称**：4字精炼表达
- **模块类型**：从7种类型中选择
- **模块内容**：具体描述（品牌、参数、技术细节等）

**配置格式示例：**
```json
{

  "name": "品牌阵列",
  "type": "brand-array",
  "content": "12个品牌对比：优选、备选、对比信息..."

```

**完整配置示例：**

  "style": "laboratory-manual",
  "modules": [
    {
      "id": "A-01",
      "name": "品牌阵列",

### 步骤 6️⃣：生成模块图像

**调用脚本生成Prompts：**

使用 `scripts/infographic_generator.py` 脚本生成各模块的图像Prompt：

```bash
python scripts/infographic_generator.py --config module_config.json --output final_infographic.png
```

脚本会：
1. 读取模块配置文件（JSON格式）
2. 为每个模块生成专门的Prompt
3. 保存Prompts到 `final_infographic_prompts.json` 文件

**配置文件格式：**

创建 `module_config.json` 文件，格式参考 `references/module-config-example.json`：

```json
{
  "title": "主题名称",
  "style": "laboratory-manual",
  "modules": [
    {
      "id": "A-01",
      "name": "品牌阵列",
      "type": "brand-array",
      "content": "具体内容描述..."
    }
  ]
}
```

**使用智能体生成模块图像：**

读取生成的Prompts文件，为每个模块生成图像：

```javascript
// 读取Prompts
const prompts = JSON.parse(readFileSync('final_infographic_prompts.json'));

// 为每个模块生成图像
for (const module of prompts) {
  generate_image(images=[{
    prompt: module.prompt,
    count: 1,
    size: {width: 800, height: 600}
  }]);
}
```

**图像命名规范：**
- 格式：`{坐标编号}_{模块名称}.png`
- 示例：`A-01_品牌阵列.png`、`B-05_风味刻度.png`

### 步骤 7️⃣：合并信息长图

**调用脚本合并图像：**

使用 `scripts/infographic_generator.py` 脚本合并已生成的模块图像：

```bash
python scripts/infographic_generator.py --output final_infographic.png --images "A-01_品牌阵列.png,B-05_风味刻度.png,..."
```

脚本会：
1. 按顺序加载所有模块图像
2. 计算总高度和最大宽度
3. 创建统一背景（浅灰色网格）
4. 垂直拼接图像，添加分隔线
5. 保存最终信息长图

**最终输出：**
- 文件名：`final_infographic.png`
- 尺寸：宽度800px，高度根据模块数量自动计算
- 格式：PNG，高质量（95%）

## 脚本使用说明

### 脚本功能

`scripts/infographic_generator.py` 提供以下功能：

1. **生成模块Prompts**：根据模块配置生成各模块的图像Prompt
2. **合并图像**：将多张模块图像合并为信息长图

### 依赖安装

```bash
pip install Pillow
```

### 参数说明

**生成Prompts模式：**
```bash
python scripts/infographic_generator.py --config module_config.json --output final_infographic.png
```

- `--config`：模块配置文件路径（JSON格式）
- `--output`：输出文件路径（用于生成Prompts文件名）

**合并图像模式：**
```bash
python scripts/infographic_generator.py --output final_infographic.png --images "image1.png,image2.png,..."
```

- `--output`：输出图像路径
- `--images`：模块图像文件路径列表（逗号分隔）

### 模块类型详解

脚本内置了7种模块类型的Prompt模板：

1. **brand-array（品牌阵列）**
   - 网格布局，展示品牌对比
   - 最佳选择用粉色高亮
   - 适合：品牌推荐、产品对比

2. **specs-scale（核心参数）**
   - 技术刻度，展示参数对比
   - 标尺或仪表盘形式
   - 适合：规格对比、等级划分

3. **deep-dive（结构拆解）**
   - 爆炸图，展示内部结构
   - 放大圈标注细节
   - 适合：产品解析、原理说明

4. **scenario-grid（场景应用）**
   - 对比卡片，展示使用场景
   - 细线分隔各场景
   - 适合：应用场景、使用方法

5. **warning-zone（避坑指南）**
   - 高对比度，展示警告信息
   - 粉色+黑色配色
   - 适合：注意事项、常见误区

6. **quick-check（快速检查）**
   - 数据表格，展示参数总结
   - 实验室数据表风格
   - 适合：参数汇总、快速参考

7. **status-bar（状态栏）**
   - 信息块，展示元数据
   - 时间戳、参数信息
   - 适合：补充信息、技术参数

### 统一视觉风格

所有模块遵循统一的视觉规范：

**色彩方案：**
- 背景：浅灰色 + 淡淡的网格纹理 (#F2F2F2)
- 主色：柔和的鼠尾草绿 (#B8D8BE)
- 强调色：荧光粉 (#E91E63) - 用于警告或高亮
- 高亮色：柠檬黄 (#FFF200) - 用于关键词
- 线条：超细炭黑 (#2D2926)

**字体风格：**
- 标题：粗体中文字符，高冲击力
- 正文：专业无衬线字体
- 数字：大号数字，黄色或蓝色高亮

**图像尺寸：**
- 单个模块：800x600 像素
- 最终长图：800px宽度，高度自动计算

## 图像生成失败处理

### 失败场景识别

当以下情况发生时，判定为图像生成失败：
- 图像生成工具返回错误
- 生成的图像完全不符合要求（如空白、严重失真）
- 图像内容与模块类型不符
- 连续2次尝试同一模块失败

### 处理流程

**模块级别降级：**

1. **单个模块失败**
   - 简化该模块的Prompt
   - 减少技术术语和细节描述
   - 重试生成

2. **重试失败**
   - 跳过该模块，继续生成其他模块
   - 记录失败模块信息
   - 向用户说明跳过的模块

3. **多个模块失败**
   - 评估整体影响
   - 提供部分完成的版本
   - 建议用户调整配置后重试

### 调试建议

**如果模块图像质量不佳：**
- 减少模块内容描述的长度
- 简化对该模块类型的特定要求
- 使用更通用的视觉描述

**如果模块图像生成超时：**
- 缩短Prompt长度
- 减少对细节的要求
- 分批生成，避免同时处理太多模块

**如果文字不清晰：**
- 增加图像尺寸（修改为1024x768）
- 减少单模块的信息密度
- 使用更大的字体描述

## 注意事项

1. **智能体能力**：本 Skill 结合智能体的自然语言理解、图像生成能力和脚本工具使用能力。

2. **模块化设计**：将高密度信息拆分为多个模块，每个模块专注单一信息维度，确保文字清晰。

3. **脚本依赖**：图像生成和合并需要Python脚本支持，依赖Pillow库。

4. **视觉一致性**：所有模块必须遵循统一的视觉语言和坐标系统，避免风格割裂。

5. **信息密度**：每个模块的信息密度适中，确保文字清晰可读，整体信息丰富。

6. **专业性要求**：确保所有数据准确、术语专业、对比合理。

7. **交互节奏**：在生成配置后主动向用户确认，确保满足需求后再执行图像生成。

8. **迭代优化**：如果用户对生成的内容或图像提出修改意见，基于反馈快速调整对应模块。

9. **失败处理**：当图像生成失败时，执行模块级别降级策略，必要时跳过模块并向用户说明。

10. **文件管理**：生成的中间文件（模块图像、Prompts文件）需妥善命名，便于后续处理。
