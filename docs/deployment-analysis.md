# 海盟会 - 部署架构与成本分析

## 一、当前运行环境

### 1. 架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                    Coze Coding 云端沙箱环境                       │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────────────┐  │
│  │ Next.js     │    │ Supabase    │    │ 火山引擎服务         │  │
│  │ 应用服务    │───▶│ PostgreSQL  │    │ - TOS 对象存储      │  │
│  │ (单实例)    │    │ (RLS)       │    │ - Seedance 视频生成 │  │
│  └─────────────┘    └─────────────┘    │ - LLM API           │  │
│         │                              │ - 知识库            │  │
│         ▼                              └─────────────────────┘  │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ Coze Integration (coze-coding-dev-sdk)                      ││
│  │ - AI 服务统一接入                                            ││
│  │ - 自动鉴权和调用                                             ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```

### 2. 当前环境特点

| 维度 | 说明 |
|------|------|
| **应用服务器** | Coze Sandbox，单实例，共享资源 |
| **数据库** | Supabase (火山引擎托管)，已有实例 |
| **存储** | TOS 对象存储 (hmhv 桶)，按量计费 |
| **AI 服务** | 通过 Coze Integration 调用，按量计费 |
| **域名** | `*.dev.coze.site` 开发域名 |

### 3. 环境限制

```
⚠️ 开发环境限制：
- 单实例运行，无负载均衡
- 共享资源，性能不保证
- 域名为开发测试域名
- 不保证 99.9% 可用性
- 适合：开发、测试、演示
- 不适合：生产环境、高并发
```

## 二、10人并发需求分析

### 1. 业务场景分析

| 场景 | 并发特点 | 资源消耗 |
|------|---------|---------|
| 用户登录/浏览页面 | 轻量级 HTTP 请求 | 低 |
| 视频生成 | 异步任务，轮询状态 | 中（API 调用） |
| 视频预览/下载 | 文件传输 | 高（带宽） |
| 智能体对话 | SSE 流式响应 | 中（长连接） |
| 学习库上传 | 文件上传 | 中（带宽 + 存储） |

### 2. 并发瓶颈分析

```
主要瓶颈：
1. 视频生成 API 限流（火山引擎）
2. 文件上传/下载带宽
3. 数据库连接数（Supabase 免费层限制）
4. 应用服务器内存/CPU

10人并发预估：
- 同时在线：10人
- 活跃请求：约 20-30 个/分钟
- 视频生成：约 5-10 个并发任务
- 文件上传：约 2-3 个并发
```

### 3. 技术方案对比

#### 方案 A：当前沙箱环境（不推荐生产）

```
✅ 优点：
- 零运维成本
- 自动部署
- 开发迭代快

❌ 缺点：
- 单点故障
- 性能不保证
- 无 SLA 保障
- 不适合正式交付

预估成本：约 0 元/月（开发环境）
风险等级：高
```

#### 方案 B：火山引擎 ECS 部署（推荐）

```
✅ 优点：
- 完全控制
- 可扩展性强
- 与现有火山服务同区域
- SLA 保障

❌ 缺点：
- 需要运维
- 迁移工作

配置推荐：
- ECS 规格：2核4G × 2（负载均衡）
- 带宽：5Mbps
- 预估成本：约 300-500 元/月
```

#### 方案 C：Serverless 部署（推荐）

```
✅ 优点：
- 按需付费
- 自动扩缩容
- 无运维
- 高可用

❌ 缺点：
- 冷启动延迟
- 调试复杂

平台选择：
- Vercel（国际）
- 阿里云函数计算
- 火山引擎 VEI（推荐，与现有服务同生态）

预估成本：约 100-300 元/月
```

## 三、费用估算（月度）

### 1. 当前架构费用

| 服务 | 计费方式 | 预估费用 |
|------|---------|---------|
| **应用服务器** | Coze Sandbox | 开发环境免费 |
| **Supabase 数据库** | 已有实例 | 包含在套餐中 |
| **TOS 对象存储** | 按量计费 | 10-50 元 |
| **Seedance 视频生成** | 按次计费 | 50-200 元 |
| **LLM API 调用** | 按 Token 计费 | 20-100 元 |
| **知识库/Embedding** | 按量计费 | 10-50 元 |
| **总预估** | | **90-400 元/月** |

### 2. 生产环境费用（推荐配置）

| 服务 | 配置 | 月费用 |
|------|------|-------|
| **应用服务器** | 2核4G × 2 + 负载均衡 | 200-300 元 |
| **Supabase** | Pro Plan（如需升级） | 0-200 元 |
| **TOS 存储** | 100GB 存储 + 流量 | 30-100 元 |
| **CDN 加速** | 视频分发 | 50-150 元 |
| **Seedance 视频生成** | 100-500 次/月 | 100-300 元 |
| **LLM API** | 对话/分析 | 50-150 元 |
| **域名 + SSL** | 自定义域名 | 10-50 元 |
| **监控 + 日志** | 基础监控 | 0-50 元 |
| **总预估** | | **440-1300 元/月** |

### 3. 费用优化建议

```
节省成本的策略：

1. 视频生成优化
   - 使用快速版模型（doubao-seedance-2-0-fast）降低成本
   - 合理设置视频时长（5秒比10秒便宜一半）
   - 缓存生成结果，避免重复生成

2. 存储优化
   - 定期清理临时文件
   - 设置视频过期策略（如30天后自动删除）
   - 使用 CDN 减少源站流量

3. AI 调用优化
   - 使用智能体双层能力系统，评分>85时减少大模型调用
   - 合理设置上下文窗口大小
   - 缓存常见问题回答

4. 数据库优化
   - 合理使用索引
   - 定期清理历史数据
   - 使用连接池
```

## 四、迁移建议

### 1. 迁移路径

```
阶段一：准备（1-2天）
├── 确认目标部署平台
├── 准备生产环境配置
├── 配置自定义域名
└── 准备 SSL 证书

阶段二：迁移（1-2天）
├── 导出数据库
├── 迁移对象存储文件
├── 配置环境变量
└── 部署应用

阶段三：测试（1天）
├── 功能测试
├── 性能测试
├── 安全测试
└── 回归测试

阶段四：上线（半天）
├── DNS 切换
├── 监控配置
├── 告警配置
└── 文档交付
```

### 2. 推荐迁移方案

**推荐方案：火山引擎 VEI（Serverless）**

理由：
1. 与现有 Supabase、TOS、AI 服务同区域，延迟最低
2. 自动扩缩容，无需运维
3. 按需付费，成本可控
4. 支持 Next.js 原生部署

## 五、移动应用开发

### 1. 技术选型

| 方案 | 技术栈 | 优点 | 缺点 |
|------|-------|------|------|
| **PWA** | Next.js PWA | 一次开发，多端运行 | 功能受限 |
| **React Native** | Expo/React Native | 跨平台，生态好 | 需要原生开发经验 |
| **Flutter** | Dart/Flutter | 性能好，UI 统一 | 学习曲线 |
| **Uni-app** | Vue/Uni-app | 国内生态好，小程序复用 | 性能一般 |

### 2. 推荐方案

**推荐：PWA + 原生壳（WebView）**

```
阶段一：PWA 优化
├── 添加 Service Worker
├── 配置离线缓存
├── 添加到桌面支持
└── 推送通知支持

阶段二：原生壳封装
├── iOS WebView 壳
├── Android WebView 壳
├── 原生功能桥接（相机、文件等）
└── 应用商店上架
```

### 3. 费用估算

| 项目 | 费用 |
|------|------|
| iOS 开发者账号 | 99 美元/年 |
| Google Play 账号 | 25 美元（一次性） |
| 应用图标/启动图设计 | 500-2000 元（可选） |
| **总预估** | **约 1000-2500 元（首年）** |
