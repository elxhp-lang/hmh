## 📌 项目概览

### 项目定位

海盟会是一个**基于 AI 的智能视频创作平台**，通过智能体辅助、用户偏好学习等创新功能，让视频创作变得简单、高效、个性化。

### 核心价值

- 🎯 **越用越懂你**：用户偏好学习系统，记录和理解你的创作习惯
- 🚀 **高效创作**：8阶段自动化工作流，从需求到视频全程自动化
- 💰 **成本优化**：批量生成、智能推荐，降低创作成本
- 🎨 **质量保证**：Seedance 2.0 标准版模型，高质量视频输出
- 🤖 **智能辅助**：智能体全程辅助，减少多模型调度时间成本
- 📊 **数据驱动**：偏好文档和习惯分析，帮助大模型更好地理解你

### 目标用户

- **内容创作者**：短视频博主、自媒体作者
- **电商卖家**：淘宝、京东、拼多多商家
- **教育机构**：在线教育、知识付费平台
- **企业市场部**：品牌宣传、产品推广
- **广告公司**：创意提案、客户服务

## ✨ 完整功能清单

### 1. 用户管理系统

#### 1.1 用户注册与登录
- ✅ **用户注册**
  - 用户名、邮箱、密码注册
  - 密码加密存储（bcrypt）
  - 注册后状态为 `pending`，需管理员审核
  - 第一个注册用户自动成为超级管理员
  - 注册成功后自动发送飞书通知给管理员

- ✅ **用户登录**
  - JWT Token 认证
  - Token 有效期 7 天
  - 审核中的用户尝试登录时显示"审核中"提示弹窗

- ✅ **密码管理**
  - 忘记密码（待实现）
  - 修改密码（待实现）

#### 1.2 权限管理
- ✅ **四级权限体系**
  - `super_admin` (超级管理员) - 权限等级 100
    - 所有权限
    - 用户管理
    - 财务数据概览
    - 查看所有素材

  - `admin` (管理员) - 权限等级 80
    - 用户管理
    - 团队数据查看
    - 审核注册用户

  - `finance` (财务) - 权限等级 60
    - 账单查看
    - 发票管理
    - 财务数据导出

  - `material_member` (普通用户) - 权限等级 20
    - 个人数据操作
    - 个人素材历史查看

- ✅ **权限分配**
  - 管理员手动分配权限
  - 权限动态审批流程

- ✅ **用户状态管理**
  - `pending` - 待审核
  - `active` - 正常
  - `disabled` - 禁用

#### 1.3 用户偏好系统（Phase 2 新增）
- ⏳ **偏好记录**
  - 记录用户选择的视频时长偏好（4秒/5秒/10秒/15秒/20秒）

  - 记录用户常用的关键词和描述方式

  - 自动生成用户偏好文档（JSON格式）
  - 文档包含：创作偏好、审美偏好、业务偏好
  - 文档版本控制，支持历史版本查看
  - 文档支持导出（JSON/Excel）

- ⏳ **习惯分析**
  - 分析用户创作时间规律（工作日/周末/时段）
  - 分析用户创作的主题类型（产品推广/品牌宣传/教程等）
  - 分析用户的创作频率和周期
  - 生成用户使用习惯报告

- ⏳ **智能应用**
  - 脚本生成时自动应用用户偏好
  - 视频参数自动匹配用户习惯

### 核心设计原则

1. **前后端分离**：清晰的 API 接口，前后端独立开发
2. **流式优先**：AI 对话采用 SSE 流式输出
3. **数据持久化**：所有生成的视频自动存储到 TOS
4. **权限控制**：基于 RLS 的行级安全策略
5. **错误处理**：统一的错误处理和日志记录
6. **可扩展性**：支持水平扩展和垂直扩展

## 🗄️ 数据库设计

### 核心数据表

#### 1. users - 用户表
```sql
CREATE TABLE users (
  id VARCHAR(36) PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(255),
  password_hash VARCHAR(255) NOT NULL,
  role VARCHAR(20) NOT NULL DEFAULT 'member', -- super_admin, admin, finance, material_member
  status VARCHAR(20) NOT NULL DEFAULT 'pending', -- pending, active, disabled
  storage_path VARCHAR(255),
  storage_quota INTEGER DEFAULT 10737418240, -- 10GB
  storage_used INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ
);
```

#### 2. videos - 视频表
```sql
CREATE TABLE videos (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  prompt TEXT NOT NULL,
  task_type VARCHAR(20) NOT NULL DEFAULT 'generate', -- generate, edit, extend
  model VARCHAR(50) NOT NULL DEFAULT 'doubao-seedance-2-0-260128',
  reference_images JSONB, -- 参考图片列表
  reference_videos JSONB, -- 参考视频列表
  reference_audios JSONB, -- 参考音频列表
  ratio VARCHAR(10) NOT NULL DEFAULT '16:9',
  duration INTEGER NOT NULL DEFAULT 5,
  status VARCHAR(20) NOT NULL DEFAULT 'pending', -- pending, processing, completed, failed
  task_id VARCHAR(100), -- Seedance 任务 ID
  tos_key VARCHAR(500), -- TOS 存储的 key
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ
);
```

#### 3. xiaohai_workflows - 创意小海工作流表
```sql
CREATE TABLE xiaohai_workflows (
  id VARCHAR(36) PRIMARY KEY,
  user_id VARCHAR(36) NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  status VARCHAR(20) NOT NULL DEFAULT 'active', -- active, paused, completed, failed
  current_step INTEGER NOT NULL DEFAULT 1, -- 1-8
  product_name VARCHAR(255),
  reference_videos JSONB,
  reference_images JSONB,
  selected_script JSONB,
  generated_scripts JSONB,
  generated_videos JSONB,
  preferred_style VARCHAR(50), -- 用户偏好风格

  preferred_duration INTEGER, -- 用户偏好时长
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ,
  completed_at TIMESTAMPTZ
);
```

## 🔒 安全保障

### 数据安全

- ✅ **行级安全策略 (RLS)**：确保用户只能访问自己的数据
- ✅ **数据加密存储**：敏感数据加密存储
- ✅ **定期数据备份**：自动备份，防止数据丢失
- ✅ **访问日志记录**：记录所有操作日志
- ✅ **敏感数据脱敏**：日志中自动脱敏敏感信息

### 权限安全

- ✅ **四级权限体系**：super_admin, admin, finance, member
- ✅ **动态权限分配**：管理员动态分配权限
- ✅ **操作日志审计**：记录所有权限操作
- ✅ **异常行为监控**：监控异常操作
- ✅ **权限过期自动回收**：定期清理过期权限

### API 安全

- ✅ **JWT Token 认证**：所有 API 需要认证
- ✅ **API 限流**：每用户每分钟最多 20 次请求
- ✅ **请求签名验证**：关键接口签名验证
- ✅ **防 SQL 注入**：使用 ORM 防止 SQL 注入
- ✅ **防 XSS 攻击**：输入输出过滤

### 网络安全

- ⏳ **HTTPS 加密**：所有传输加密（Phase 2）
- ⏳ **DDoS 防护**：防止 DDoS 攻击（Phase 2）
- ⏳ **WAF 防火墙**：Web 应用防火墙（Phase 2）
