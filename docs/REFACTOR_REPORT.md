## 架构对比

### 之前（过度设计）

```
用户输入
  ↓
8阶段工作流状态机
  ↓
双层能力系统（学习模式/对比模式/独立模式）
  ↓
能力评分系统
  ↓
复杂的状态转换逻辑
  ↓
返回结果
```

**问题**:
- 过度复杂
- 偏离本质
- 维护困难

### 现在（简单高效）

```
用户输入
  ↓
意图识别（关键词匹配）
  ↓
模型调度
  ├─ 对话模型
  ├─ 视觉模型
  └─ 脚本生成模型
  ↓
返回结果 + 记录偏好
  ↓
人格蒸馏（异步）
```

**优势**:
- 简单直接
- 易于维护
- 核心价值清晰

## 核心价值实现

### 1. 减少用户重复工作

**场景**: 用户要生成10条产品视频

**流程**:
```
第1次: 记录偏好（风格、时长、修改意见...）
第2次: 自动应用偏好，减少修改
第3-10次: 批量生成，全部符合用户风格
```

**效果**:
- 命中率: 30% → 80%（预期）
- 修改次数: 3次 → 0.5次（预期）
- 沟通轮次: 5轮 → 1轮（预期）

### 2. 人格蒸馏系统

**触发时机**:
- 用户完成一次创作
- 积累到5条以上偏好
- 发现新的偏好模式

**人格内容**:
- 创作偏好：风格、时长、平台
- 视觉偏好：镜头、色调
- 内容偏好：字幕大小、背景音乐
- 修改习惯：常见修改意见

**应用场景**:
- 批量生成
- 新的创作任务
- 减少重复沟通
