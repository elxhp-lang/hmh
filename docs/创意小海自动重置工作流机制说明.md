# 创意小海自动重置工作流机制说明

> **实施日期**: 2026-04-07
> **功能**: 自动检测对话超时，自动开启新工作流

---

## 📋 问题描述

用户反馈创意小海存在以下问题：
1. 不会主动开启新工作流
2. 进入对话框时，依然是之前的对话
3. 没有欢迎语
4. 没有重置工作流

---

## ✅ 解决方案

### 核心机制

**单工作流模式 + 超时自动重置**

- **超时时间**: 1小时
- **自动行为**:
  - 若1小时无继续对话，则自动视为结束
  - 若后续用户再次打开对话框，则自动开启新工作流
  - 自动发送欢迎语
  - 清理历史对话

### 实现方式

#### 1. 使用 localStorage 记录最后对话时间

```typescript
// 记录最后对话时间
const updateLastConversationTime = () => {
  localStorage.setItem('xiaohai_last_conversation_time', Date.now().toString());
};

// 在每次发送消息时更新
sendMessage(...) {
  updateLastConversationTime();
  // ...
}
```

#### 2. 组件挂载时检查超时

```typescript
useEffect(() => {
  // 防止重复发送欢迎语
  if (hasAutoWelcomed.current) return;

  // 获取上次对话时间
  const lastConversationTime = localStorage.getItem('xiaohai_last_conversation_time');
  const currentTime = Date.now();

  // 如果超过1小时
  const shouldReset = !lastConversationTime ||
    (currentTime - parseInt(lastConversationTime) > 60 * 60 * 1000);

  if (shouldReset) {
    // 清理状态
    setMessages([]);
    setWorkflowId('');
    setAttachments([]);

    // 自动发送欢迎语
    setTimeout(() => {
      sendMessage('welcome');
    }, 100);
  }
}, []); // 仅在组件挂载时执行一次
```

#### 3. 添加手动重置按钮（可选）

```typescript
const resetWorkflow = () => {
  setMessages([]);
  setWorkflowId('');
  setAttachments([]);
  hasAutoWelcomed.current = false;

  setTimeout(() => {
    sendMessage('welcome');
  }, 100);
};

// 快捷操作中添加
{ label: '🔄 重置工作流', action: 'reset' }
```

## 🔍 技术细节

### 防止重复欢迎

使用 `hasAutoWelcomed.current` 标志，确保：
- 组件挂载时只检测一次
- 不会因为状态更新重复触发
- 即使不重置也标记为已处理

### 状态清理

清理以下状态：
- `messages`: 历史消息
- `workflowId`: 工作流ID
- `attachments`: 附件
- `input`: 输入框内容
- `currentStep`: 当前步骤（auto-test页面）
- `uploadedFile`: 上传的文件（debug页面）
- `uploadedUrl`: 上传的URL（debug页面）

### 延迟发送欢迎语

使用 `setTimeout` 延迟100ms，确保：
- 状态清理完成后再发送欢迎语
- 避免React状态更新冲突

## 🔧 配置说明

### 修改超时时间

如需修改超时时间，找到以下代码：

```typescript
const shouldReset = !lastConversationTime ||
  (currentTime - parseInt(lastConversationTime) > 60 * 60 * 1000);
```

修改 `60 * 60 * 1000` 为所需的毫秒数：
- 30分钟: `30 * 60 * 1000`
- 2小时: `2 * 60 * 60 * 1000`
- 24小时: `24 * 60 * 60 * 1000`

### 禁用自动重置

如需禁用自动重置，注释掉超时检测的 useEffect。

## 📌 注意事项

1. **localStorage 存储**: 每个页面使用独立的 localStorage key，互不干扰
2. **组件生命周期**: useEffect 仅在组件挂载时执行一次，不会因为状态更新重复触发
3. **防止重复**: 使用 `hasAutoWelcomed.current` 标志确保只检测一次
4. **延迟执行**: 使用 setTimeout 确保状态清理完成后再发送欢迎语
5. **向后兼容**: 不影响现有的工作流功能
