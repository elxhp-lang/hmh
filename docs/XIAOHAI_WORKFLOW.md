# 创意小海工作流设计文档

## 核心要求

### 脚本格式
- **时间维度**：3秒为单位
- **内容要素**：
  - 画面描述
  - 人物（包含面部细节）
  - 动作
  - 口播内容
  - 整体风格

## 完整工作流（8个步骤）

### 前置条件
- 脚本以3秒为时间维度
- 包含：画面、人物（面部细节）、动作、口播、风格

---

### 步骤1：唤醒与偏好展示

**触发**：用户打开页面

**动作**：
1. 小海发送欢迎语
2. 展示功能介绍
3. 展示用户偏好总结
4. 询问是否依据此偏好开始工作

**示例**：
```
🌊 你好！我是创意小海

**我可以帮你**：
- 分析视频风格并创作脚本
- 根据商品库制作产品视频
- 批量生成同风格视频

**你的创作偏好**：
- 风格：自然真实、生活化
- 时长：15秒
- 平台：抖音
- 镜头：推镜、固定镜头
- 字幕：大字号

是否依据此偏好开始今天的工作？
```

---

### 步骤2：任务识别与确认

**用户输入类型**：
1. 上传视频文件
2. 平台视频链接（抖音、快手等）
3. 商品名称（"生成XX商品的视频"）

**小海动作**：
1. 识别用户意图
2. 发送任务接受语句
3. 明确工作内容

**示例**：
```
✅ 已收到你的需求！

我理解你要：
- 分析抖音视频：[视频链接]
- 制作产品：保温杯
- 生成风格：自然真实、生活化

正在开始处理...
```

---

### 步骤3：模型调用与进度反馈

**场景A：用户上传视频/平台链接**
1. 调用视觉分析模型（doubao-seed-1-6-vision）
2. 保存视频到 TOS
3. 作为参考视频（供 Seedance 2.0 使用）

**场景B：用户要求生成商品库产品**
1. 直接调用脚本生成模型
2. 无需视频分析

**进度反馈**：
```
📊 正在分析视频...

当前工作：
- 模型：doubao-seed-1-6-vision
- 任务：视频风格分析
- 预计耗时：30秒

[进度条: ████████░░ 80%]
```

**用户中断处理**：
- 识别新需求：修改参数 / 发起新任务
- 询问确认：是否暂停当前任务
- 保存当前进度

---

### 步骤4：脚本呈现与确认

**输出内容**：
1. 参考脚本（分析结果）
2. 保存到用户脚本数据库
3. 蒸馏用户偏好

**用户选择**：
- 保持脚本不变
- 微调二创

**产品确认**：
```
✅ 视频分析完成！

**参考脚本**：
| 时间 | 画面 | 人物 | 动作 | 口播 | 风格 |
|------|------|------|------|------|------|
| 0-3秒 | 厨房晨光 | 女生，扎马尾，素颜 | 搅拌粥 | "早安" | 暖色调 |

请问：
1. 脚本是否需要微调？
2. 本次制作的产品是？（从商品库选择）
```

**生成3个类似脚本**：
- 带上蒸馏后的用户偏好
- 调用脚本生成模型
- 输出3个不同风格

---

### 步骤5：脚本生成进度反馈

**状态反馈**：
- 未完成：发送"脚本创作中，请稍等"
- 已完成：发送"新脚本已完成" + 3个脚本

**示例**：
```
📊 脚本创作中...

已完成：0/3
[进度条: ██░░░░░░░░ 20%]
```

```
✅ 新的类似脚本已完成！

脚本1：[内容]
脚本2：[内容]
脚本3：[内容]

是否开始短视频制作？
```

---

### 步骤6：视频生成

**触发**：用户确认开始制作

**输入**：
1. 商品外观图片（商品库）
2. 脚本
3. 参考视频（用户上传/视频1）

**调用**：Seedance 2.0（多模态参考）

**进度反馈**：
```
🎬 视频制作中...

当前工作：
- 模型：Seedance 2.0
- 任务：视频生成
- 预计耗时：2分钟

[进度条: ██████░░░░ 60%]
```

**完成**：
```
✅ 视频1 已完成！

[视频预览]

下一步：
- 批量制作类似视频
- 结束本次创作
```

---

### 步骤7：批量生成

**流程**：
1. 自动执行：分析 → 生成脚本 → 制作视频
2. 参考视频：用户上传视频 + 视频1 + 视频2
3. 多模态参考限制：最多3个视频

**循环逻辑**：
- 视频2：用户上传视频 + 视频1
- 视频3：用户上传视频 + 视频1 + 视频2
- 视频4-N：用户上传视频 + 视频1 + 视频2（固定参考）

**完成询问**：
```
✅ 视频2 已完成！

[视频预览]

是否继续批量生成？
- 继续制作
- 结束本次创作
```

---

### 步骤8：结束与保存

**动作**：
1. 保存对话信息
2. 蒸馏用户偏好
3. 结束工作流

**状态**：
- 工作流不可二次启用
- 用户偏好持续积累

---

## 特殊场景处理

### 1. 多工作流并行

**场景**：用户在视频生成时发起新任务

**处理**：
1. 保存当前工作流进度
2. 开启新工作流
3. 工作流隔离
4. 支持切换（通过产品名/脚本关键词）

**示例**：
```
用户：帮我再做一个收纳箱的视频

小海：
✅ 已暂停当前工作流（保温杯视频）
✅ 新工作流已启动（收纳箱视频）

你可以随时说"切换到保温杯"继续之前的工作
```

### 2. 违规视频处理

**场景**：用户上传真人视频

**检测**：Seedance 2.0 不支持真人人脸

**处理**：
```
⚠️ 检测到参考视频包含真人画面

Seedance 2.0 不支持真人视频生成

建议：
- 直接根据产品图和脚本创作
- 上传产品展示视频作为参考
```

## 数据结构设计

### 工作流状态

```typescript
interface Workflow {
  id: string;
  userId: string;
  status: 'active' | 'paused' | 'completed';
  currentStep: 1-8;
  
  // 任务信息
  taskType: 'video_analysis' | 'product_generation';
  product?: string;
  referenceVideos: string[]; // 最多3个
  
  // 脚本
  scripts: Script[];
  selectedScriptIndex?: number;
  
  // 视频生成
  generatedVideos: string[];
  
  // 进度
  progress: {
    currentModel: string;
    currentTask: string;
    percentage: number;
    estimatedTime: number;
  };
  
  // 偏好蒸馏
  preferences: UserPreference[];
  
  // 时间戳
  createdAt: Date;
  updatedAt: Date;
}
```

### 脚本格式

```typescript
interface Script {
  id: string;
  scenes: Scene[];
  style: string;
  duration: number;
}

interface Scene {
  time: string; // "0-3秒"
  visual: string; // 画面
  character: string; // 人物（面部细节）
  action: string; // 动作
  voiceover: string; // 口播
  style: string; // 风格
}
```

- 3秒时间维度
- 必须包含人物面部细节

- 显示预计耗时
- 进度条可视化

- 无需用户干预

- 持续积累
- 应用于后续生成
